# 高尔夫追球外部研究报告

生成日期：2026-07-17

范围：高速微小球检测、时序跟踪、遮挡恢复、数据与训练、单目/多目轨迹、飞行动力学、TrackMan 约束和端侧部署。
方法：优先检索论文原文、会议论文页、作者代码仓库和官方产品/开发文档；博客与论坛不作为核心结论依据。

## 执行摘要

高尔夫追球不是普通目标检测问题，而是“小目标 + 高相对速度 + 运动模糊 + 可见性变化 + 单目尺度不确定 + 严格物理约束”的组合问题。外部研究和本项目失败证据共同指向一个混合架构：高分辨率时序热图模型负责召回；直接像素证据和状态机负责逐点真实性；Kalman/Viterbi/点跟踪器只负责搜索、排序和短时预测；相机几何与 RK4 飞行动力学负责 3D 一致性；最终接受仍由逐帧证据和人工审核决定。

最重要的外部结论有五点：

1. TrackNet 证明连续帧热图比单帧 bbox 更适合高速、模糊、带拖影的微小球，但跨场景验证的召回明显低于单场景结果，说明域泛化是核心风险。
2. WASB-SBDT 将高分辨率特征、位置感知训练和时序一致性组合成跨运动类别基线，直接支持本项目的 high-resolution + temporal-evidence 路线。
3. TSFMO 对 20 种跟踪器的评估显示，通用跟踪榜单不能代表小快目标能力；其高尔夫序列也暴露出低层纹理和高层语义需要联合使用。
4. TAPIR、CoTracker/CoTracker3 对遮挡与再出现很有价值，但它们追踪“查询点对应关系”，并不天然知道该点是否仍是高尔夫球，因此只能作为候选或反证。
5. 高尔夫弹道需要速度相关阻力、旋转相关升力/Magnus 和数值积分；物理与深度学习混合模型适合估计不可直接观测的发射状态，但物理拟合不能把不可见帧升级成训练真值。

## 1. 高速微小球检测

### 1.1 TrackNet 系列

TrackNet 使用单帧或连续帧输入生成球位置热图，专门针对小、模糊、拖影甚至暂时不可见的高速球。原论文的单场景测试非常高，但扩展视频交叉验证召回下降，说明场地、背景、曝光和球外观变化必须进入训练划分和验证设计。[TrackNet 论文](https://arxiv.org/abs/1907.03698)

TrackNetV3 在 TrackNet 系列基础上增加增强与轨迹修复；其公开实现明确处理遮挡后的路径修复。但对 golf_15 而言，“修复轨迹”只能产生 `occluded/predicted` 状态，不能把补出的点直接作为 visible center。[TrackNetV3 论文](https://people.cs.nycu.edu.tw/~yushuen/data/TrackNetV3.pdf)；[作者代码](https://github.com/qaz812345/TrackNetV3)

### 1.2 WASB-SBDT

WASB-SBDT 的三项核心设计是高分辨率特征提取、位置感知训练和考虑时序一致性的推理，并在五类运动数据上与六种方法比较。对本项目最直接的启发是：不要过早下采样；训练采样要体现球可能出现的位置；时序推理必须保留 MISS/遮挡，而不是强制每帧出点。[BMVC 论文页](https://proceedings.bmvc2023.org/310/)；[代码与数据](https://github.com/nttcom/WASB-SBDT)

### 1.3 高分辨率与多尺度

Feature Pyramid Networks 通过自顶向下路径和横向连接融合高层语义与高分辨率细节，是小目标检测的经典多尺度基础。高尔夫球可能只有数个像素，模型设计应保留 P2/高分辨率分支或使用局部 ROI，而不是只依赖深层低分辨率特征。[FPN 论文](https://openaccess.thecvf.com/content_cvpr_2017/html/Lin_Feature_Pyramid_Networks_CVPR_2017_paper.html)

## 2. 跟踪、遮挡和再捕获

### 2.1 小快目标基准的警告

TSFMO 包含 26 类运动、250 段视频和约 5 万帧，并评估 20 种跟踪器。论文报告其目标平均面积更小、相对速度更高，通用 benchmark 上排名靠前的跟踪器在 TSFMO 上并不一定可靠；高尔夫序列也需要低层与高层特征融合。[TSFMO 论文](https://openaccess.thecvf.com/content/ACCV2022/papers/Zhang_Tracking_Small_and_Fast_Moving_Objects_A_Benchmark_ACCV_2022_paper.pdf)；[代码](https://github.com/CodeOfGithub/S-KeepTrack)

### 2.2 光流、Kalman 与配准

OpenCV 的金字塔 Lucas–Kanade 光流适合在短时间、局部亮度近似不变时估计点运动；快速模糊、外观消失和大位移会破坏这些假设。Kalman filter 可利用真实帧间隔传播位置和速度，并通过协方差表达不确定性。`findTransformECC` 是基于强度相似性的区域配准，不是球身份判定器。[OpenCV tracking 文档](https://docs.opencv.org/4.x/dc/d6b/group__video__track.html)；[KalmanFilter 文档](https://docs.opencv.org/4.x/dd/d6a/classcv_1_1KalmanFilter.html)；[光流教程](https://docs.opencv.org/4.x/d4/dee/tutorial_optical_flow.html)

### 2.3 TAPIR 与 CoTracker

TAPIR 采用逐帧匹配和时序精修两阶段结构，并显式处理可见性；CoTracker 联合跟踪大量点以利用点之间依赖；CoTracker3 用真实视频伪标签改善 sim-to-real。它们适合提供“这个 seed 可能移动到哪里”的研究候选，但高尔夫球在模糊、遮挡后可能与杆头、头发、灯光或背景纹理发生身份切换，所以必须再经过球特定像素门禁。[TAPIR](https://openaccess.thecvf.com/content/ICCV2023/html/Doersch_TAPIR_Tracking_Any_Point_with_Per-Frame_Initialization_and_Temporal_Refinement_ICCV_2023_paper.html)；[CoTracker](https://arxiv.org/abs/2307.07635)；[CoTracker3](https://openaccess.thecvf.com/content/ICCV2025/papers/Karaev_CoTracker3_Simpler_and_Better_Point_Tracking_by_Pseudo-Labelling_Real_Videos_ICCV_2025_paper.pdf)

## 3. 数据、标注与评估

TrackNet 的数据设计区分可见、模糊/拖影、遮挡和事件状态，并利用相邻帧辅助人工理解。对 golf_15，应继续区分 `visible observed`、`occluded predicted` 和 `lost`，但只有直接可见且人工确认/规则验收的中心点进入训练。

SoccerNet-Tracking 提供完整序列和 track ID，并指出快速运动与严重遮挡仍远未解决。它提醒本项目不能只按随机帧切分数据，必须按 session/shot/video 分组，避免相邻帧泄漏。[SoccerNet-Tracking](https://openaccess.thecvf.com/content/CVPR2022W/CVSports/papers/Cioppa_SoccerNet-Tracking_Multiple_Object_Tracking_Dataset_and_Benchmark_in_Soccer_Videos_CVPRW_2022_paper.pdf)

建议评估层次：

- 点级：像素误差、PCK、precision/recall、visibility accuracy。
- 序列级：observed coverage、最大缺口、身份切换、折返、跳点、静止锁、再捕获正确率。
- 场景级：球尺寸、速度、模糊、遮挡、天空/草地/灯光、曝光和杆头邻近分层。
- 物理级：重投影误差、发射速度/角度、自旋、carry/apex/side 的误差与校准区间。
- 系统级：端到端延迟、内存、功耗、黑屏/缺帧率和人工审核时间。

## 4. 3D 几何与飞行动力学

### 4.1 观测模型

针孔投影为：

`X_c = R X_w + t`

`u = f_x X_c / Z_c + c_x`

`v = f_y Y_c / Z_c + c_y`

单个像素只确定一条射线，不能唯一给出深度。单目 3D 必须引入相机标定、初始球位置/地面、时间连续性、已确认 TrackMan 参数或飞行动力学等独立约束。

### 4.2 飞行方程

设相对空气速度 `v_r = v - w`：

`dr/dt = v`

`dv/dt = g - (ρ C_D A / 2m) ||v_r|| v_r + a_M`

Magnus 项可写成：

`a_M = (ρ C_L A / 2m) ||v_r||² f_spin normalize(v̂_r × ω̂)`

高尔夫球的阻力和升力系数会随 Reynolds 数、自旋参数和球面结构变化；固定常数是工程近似而非普适真值。USGA 教学资料明确强调 drag 与 lift 对球飞行的作用；实验和轨迹研究也显示球面和自旋会改变气动系数。[USGA aerodynamics](https://www.usga.org/content/dam/usga/pdf/science-of-golf/middle-school/The%20Ball/aerodynamics_facilitator_guide_MS.pdf)；[Aerodynamics of Golf Balls in Still Air](https://www.mdpi.com/2504-3900/2/6/238)

### 4.3 数值积分与混合模型

RK4 每步为：

`k1=f(s_n)`

`k2=f(s_n+Δt k1/2)`

`k3=f(s_n+Δt k2/2)`

`k4=f(s_n+Δt k3)`

`s_(n+1)=s_n+Δt(k1+2k2+2k3+k4)/6`

CVPR Sports 论文研究了物理模型与深度学习结合模拟高尔夫球飞行，支持“网络估计难观测参数 + 物理积分保证结构”的路线。[Physics + Deep Learning Golf Flight](https://openaccess.thecvf.com/content/CVPR2023W/CVSports/papers/McNally_Combining_Physics_and_Deep_Learning_Models_To_Simulate_the_Flight_CVPRW_2023_paper.pdf)

## 5. TrackMan 与多传感器约束

TrackMan 官方说明 TrackMan 4 结合双雷达和相机，提供多类球杆与球数据，并可覆盖完整飞行。对本项目而言，TrackMan 是重要但需要人工确认时间匹配、shot 身份和结构化字段的外部证据；OCR 文本或未匹配照片不能直接进入误差计算。[TrackMan 4 官方页](https://www.trackman.com/golf/launch-monitors/trackman-4)

建议把 TrackMan 分成三层：

1. 原始证据：照片/导出、时间戳和来源文件。
2. review 层：shot 匹配、时区、delta、人工纠错和未匹配项。
3. 约束层：只有 confirmed 结构化 ball speed、launch angle/direction、spin、carry/apex/side 可进入损失或验收。

## 6. 模型训练与自训练

推荐训练路线：

1. 用合成球、运动模糊、曝光扰动和真实背景预训练高分辨率热图模型。
2. 用 V1.2 人工确认标签进行真实微调，并按 session/shot 分组切分。
3. 对小球尺寸、极高速、遮挡、杆头邻近、天空和强光建立难例桶。
4. 模型输出始终是 prelabel；逐点像素证据、轨迹 QC 和人工 review 通过后才能进入下一轮训练。
5. 伪标签训练可借鉴 CoTracker3，但 teacher agreement 不能替代人工标签边界。

推荐消融：单帧 vs 5 帧；低分辨率 vs 高分辨率；global vs ROI；无/有运动注意力；无/有 hard-negative；无/有 visibility head；Kalman vs Viterbi；通用点跟踪候选开/关；物理损失开/关。

## 7. 端侧部署

Apple 官方说明 Core ML 会使用 CPU、GPU 和 Neural Engine，并面向低内存、低功耗的设备端推理。WWDC24 还提供 MLComputePlan 与性能报告等分析能力。部署验收不能只看模型转换成功，还应验证输入预处理、输出坐标、数值 parity、热图峰值、连续帧状态、延迟、峰值内存和真机热状态。[Core ML 官方文档](https://developer.apple.com/documentation/coreml)；[WWDC24 Core ML 部署](https://developer.apple.com/videos/play/wwdc2024/10161/)

## 8. 对 golf_15 的决策建议

### 立即采用

- 高分辨率时序热图 + offset + visibility 作为正式模型主干。
- 全局模型负责召回，局部 ROI 模型负责精修；二者输出统一证据契约。
- 逐点 direct-pixel evidence 与轨迹状态机作为任何自动点的硬门禁。
- 按 session/shot/video 分组切分和场景分层评估。
- V47 四类反例固化为回归集。

### 研究性采用

- TAPIR/CoTracker3 作为搜索和反证 provider，单独统计身份切换率。
- WASB 的高分辨率模块、位置感知训练和时序推理做定量消融。
- 物理 + 深度学习混合发射状态估计，但只用独立监督训练。
- Core ML 状态模型或滑动窗口模型在真机比较延迟、内存和稳定性。

### 明确禁止

- 把 Kalman、Viterbi 插值、TAPIR/CoTracker 或物理轨迹直接写成 visible label。
- 只以点数/coverage 增长判定进步。
- 用页面无黑屏证明轨迹语义正确。
- 用未确认 TrackMan/OCR 参与误差或训练。
- 用 Bezier、样条或 endpoint interpolation 替代 RK4 + drag + Magnus。

## 研究限制

- 高尔夫球专用、开放、逐帧人工球心数据集仍明显少于网球/羽毛球数据集。
- TrackMan 的公开产品资料不能替代本项目自身的时间匹配与误差标定实验。
- 不同论文的数据、帧率、目标大小和评价指标不可直接横向比较；必须在 golf_15 同一 split 上复现。
- 2025—2026 新点跟踪和模糊球研究值得继续观察，但在完成本项目反例集验证前不应替换现有主线。
